## 代理节点来源

```
proxy-providers (机场订阅)
  ├── Airport_01  [BOOST]  boost1.shop
  └── Airport_02  [bajie]  q1w2d3.org
         │
         │  filter: 过滤掉广告、流量信息等无效节点
         │
         ▼
    所有代理节点 (include-all: true 引入)
```

两个机场订阅通过 `PProviders` 锚点共享配置，过滤掉含"拒绝/直连/流量/到期"等关键词的信息节点。

### 各层说明

**服务分流层** — 直接在 UI 上操作，为特定服务选择路由策略

| 组名 | 类型 | 默认 | 说明 |
|------|------|------|------|
| 节点选择 | select | 香港节点 | 所有走代理的流量默认经过这里 |
| AI | select | 节点选择 | ChatGPT/Claude 等 AI 服务，可选择所有节点 |
| Apple | select | 全球直连 | Apple 服务，默认直连 |
| Emby | select | 节点选择 | Emby 媒体服务 |
| STEAM | select | 节点选择 | Steam 游戏平台 |
| 全球直连 | select | 直连 | 直连流量，可切换为走代理 |
| 隐私拦截 | select | 拒绝 | 广告/追踪，可选拒绝/丢弃/直连/代理 |
| Final | select | 节点选择 | 所有未匹配规则的流量兜底 |

**区域节点层** — 按地区分组，手动选择具体节点

| 组名 | 筛选关键词 | 特殊选项 |
|------|-----------|---------|
| 香港节点 | 港/hk/hongkong | 自动+均衡 |
| 新加坡节点 | 新/SG/singapore | 自动+均衡 |
| 日本节点 | 日本/JP/Japan | 自动+均衡 |
| 台湾节点 | 台/TW/Taiwan | 自动+均衡 |
| 美国节点 | 美/US/America | 自动+均衡，排除澳/俄 |
| 欧洲节点 | 各欧洲国旗 emoji | 无自动/均衡 |
| 自建/家宽节点 | 自建/CF/private/家宽 | 家宽/自建节点 |
| 全部节点 | 无筛选 | 所有节点手动选 |

**自动/均衡层** (hidden，不直接在 UI 显示)

| 组名 | 类型 | 作用 |
|------|------|------|
| xx自动 | url-test | 自动选延迟最低的节点，容差 20ms |
| xx均衡 | load-balance | consistent-hashing 负载均衡 |

**故障转移** — fallback 类型，按顺序检查：香港→新加坡→日本→台湾→美国→欧洲→自建，当前节点不可用时自动切换。

  ├── Steam ─────────────────→ STEAM 组

## DNS 解析流程

```
应用发起 DNS 请求
       │
       ▼
  TUN 劫持 (any:53)
       │
       ▼
  mihomo DNS 模块 (fake-ip 模式)
       │
       ├── 匹配 fake-ip-filter?
       │     (geosite:cn / category-games@cn / private / rule-set)
       │       │
       │    是 → 返回真实 IP (用直连 DNS 解析)
       │       │
       │       ├── direct-nameserver: doh.pub / 223.5.5.5
       │       │
       │    否 → 返回 fake-ip (28.0.0.0/8 段)
       │
       ▼
  实际连接时解析域名
       │
       ├── 走代理的流量 → nameserver (dns.google / cloudflare) #RULES
       │                   通过代理节点发出 DNS 查询
       │
       ├── 走直连的流量 → direct-nameserver (doh.pub / 223.5.5.5)
       │
       └── 代理节点域名 → proxy-server-nameserver (doh.pub / 223.5.5.5)
                           用国内 DNS 解析节点服务器地址，避免循环依赖
```

## 锚点复用说明

配置使用 YAML 锚点 (`&`/`<<:*`) 减少重复：

| 锚点 | 用途 | 被引用次数 |
|------|------|-----------|
| `Direct_first` | 直连优先的 select 组 | Apple |
| `Include_all` | 包含所有节点的 select 组 | AI, Emby, STEAM, Final |
| `Urltest_Base` | 自动测速的基础配置 | 5 个区域自动组 |
| `PProviders` | 机场订阅基础配置 | 2 个 provider |
